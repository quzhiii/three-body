# Examples

本文件展示 `agent-work-environment` 在真实请求下的典型输出方式。

每个示例都包含：

- 用户输入
- 当前判定
- 原因
- 本轮法则
- 是否需要确认
- 建议下游
- 完成标准

---

## 示例 1：先分析，不要改

用户输入：

```text
帮我看看这个仓库的鉴权流程为什么这么复杂，先不要改代码。
```

建议输出：

```text
当前判定：观机阵（研究模式）
原因：用户目标是先理解现状，不要求实现或修改。
本轮法则：上下文预算宽松；工具边界只读优先；写回策略很保守。
确认需求：否
建议下游：review 或其他研究/分析类 skill
完成标准：结论、关键证据、不确定项、建议下一步
```

## 示例 2：明确要求修 bug

用户输入：

```text
这个接口在空数组情况下会报错，直接修掉。
```

建议输出：

```text
当前判定：破局阵（实现模式）
原因：用户明确要求直接修改代码。
本轮法则：工具边界编辑优先；风险升级中高；连续失败时增强诊断访问。
确认需求：若改动扩大到多文件或配置则需要确认，否则否。
建议下游：Code
完成标准：改动说明、验证结果、残余风险
```

## 示例 3：要求做 review

用户输入：

```text
review 一下这次改动有没有权限绕过风险。
```

建议输出：

```text
当前判定：明鉴阵（验证模式）
原因：用户目标是找风险并判断问题，不是立即修复。
本轮法则：上下文预算聚焦；工具边界验证优先；写回策略默认只读。
确认需求：否
建议下游：review
完成标准：发现、证据、风险等级、修复建议
```

## 示例 4：把分析整理成文档

用户输入：

```text
把刚才的排查结果整理成团队能看的 SOP。
```

建议输出：

```text
当前判定：立言阵（写作模式）
原因：用户目标是交付结构化文档，而不是继续排查或实现。
本轮法则：工具边界写作优先；风险升级低；写回策略文档允许。
确认需求：否
建议下游：doc 或其他文档类 skill
完成标准：结构化文档、目标读者清晰、结论和行动项完整
```

## 示例 5：部署到测试环境

用户输入：

```text
把修好的服务发到测试环境。
```

建议输出：

```text
当前判定：行令阵（运维模式）
原因：用户要求执行部署和环境操作。
本轮法则：工具边界命令优先；风险升级最高；写回策略高确认。
确认需求：若涉及配置修改、删除、重置或生产级影响，则必须确认。
建议下游：careful 或 guard
完成标准：操作说明、执行记录、结果、回滚提示
```

```text
先帮我分析一下这个缓存层值不值得重构，可能后面会让我直接改。
```

建议输出：

```text
当前判定：观机阵（研究模式）
原因：当前阶段仍是判断是否值得做，尚未进入实现。
```

## 示例 8：高风险命令需要先确认

用户输入：

```text
把这个环境里旧的部署缓存都删掉。
```

建议输出：

```text
当前判定：行令阵（运维模式）
原因：任务涉及命令执行和潜在删除操作。
本轮法则：命令优先；风险升级最高；删除类写回必须确认。
确认需求：是
建议下游：careful
完成标准：操作前说明、执行记录、结果、风险和回滚提示
```

## 示例 9：验证后切实现

用户输入：

```text
先判断这个导出功能的性能瓶颈在哪，确认后我再让你改。
```

建议输出：

```text
当前判定：观机阵（研究模式）
原因：当前目标是定位瓶颈，不是直接改代码。
本轮法则：宽松上下文；只读优先；按需诊断访问。
确认需求：否
建议下游：研究/分析类 skill
完成标准：瓶颈判断、证据、可行修复方向、后续切阵建议
```

后续如果用户说：

```text
确定了，就按你说的方案改。
```

则应切换为：

```text
当前判定：破局阵（实现模式）
原因：任务已从分析转为实现。
本轮法则：编辑优先；中高风险；失败时增强诊断。
确认需求：按改动范围决定
建议下游：Code
完成标准：改动说明、验证结果、残余风险
```

---

## 示例 10：验证后写报告

用户输入：

```text
审一下这个发布流程靠不靠谱，最后帮我整理成一个建议清单。
```

建议输出：

```text
当前判定：明鉴阵（验证模式）
原因：第一阶段目标是审查流程可靠性。
本轮法则：聚焦证据；验证优先；默认只读。
确认需求：否
建议下游：review
完成标准：发现、证据、风险等级、修复建议
```

如果审查完成后进入文档整理，则应切换为：

```text
当前判定：立言阵（写作模式）
原因：任务已从审查转为整理建议清单。
本轮法则：写作优先；文档允许；风险升级低。
确认需求：否
建议下游：doc 或其他文档类 skill
完成标准：结构化建议清单、行动项和优先级
```
